# Tournament Predictor

一个用于预测 16 支队伍比赛全过程的网页工具。

当前支持：

- `16 进 8` 瑞士轮预测
- `8 强及以后` 严格双败赛预测
- 拖拽队伍进入首轮比赛
- 双击队伍判定比赛胜者
- 自动生成后续对阵
- 队伍名称与颜色自定义编辑
- `localStorage` 本地存档

## 功能概览

- 瑞士轮第 1 轮由用户手动建局
- 瑞士轮后续轮次自动配对，直到产生 8 支晋级队伍
- 8 强阶段自动生成双败树
- 总决赛支持败者组冠军击败不败队后的加赛
- 刷新页面后可恢复本地预测进度
- 支持一键清空本地存档并重置全部预测

## 技术栈

- `React`

## 本地运行

在项目目录执行：

```powershell
npm.cmd install
npm.cmd run dev
```

然后打开终端输出的本地地址，通常是：

```txt
http://localhost:5173
```

## 构建

```powershell
npm.cmd run build
```

构建产物位于：

```txt
dist/
```

## 使用方式

### 瑞士轮

1. 从左侧队伍池拖拽队伍到首轮比赛卡。
2. 每场两支队伍齐全后，双击其中一支队伍判定胜者。
3. 系统会自动生成后续瑞士轮对阵。
4. 当产生 8 支三胜队伍后，瑞士轮结束。

### 双败赛

1. 瑞士轮结束后，系统自动生成 8 强双败树。
2. 双击任意已就位队伍判定该场胜者。
3. 系统自动推进胜者组、败者组和总决赛路径。
4. 若败者组冠军先赢总决赛，将自动出现加赛。

### 队伍编辑

1. 双击左侧任意队伍卡。
2. 在弹窗中修改队伍名字和颜色。
3. 保存后会同步更新整个预测界面。

## 数据存储

项目当前为纯前端应用。

- 所有预测数据保存在浏览器 `localStorage`
- 不依赖后端数据库
- 更换浏览器或清除浏览器数据后，本地存档会丢失

## 部署

本项目可以作为静态站点部署。

基本流程：

1. 运行 `npm.cmd run build`
2. 将 `dist/` 目录内容上传到服务器
3. 用 `Nginx`、`Apache`、`IIS` 或静态托管平台提供访问

如果使用前端路由，服务器需要将未知路径回退到 `index.html`。

## 当前限制

- 仅支持单用户本地预测
- 不支持账号系统与云端同步
- 不支持多人共享同一份预测数据
